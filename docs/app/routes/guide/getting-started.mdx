# Getting Started

Get a complete ESLint setup in under a minute.

## Installation

<Steps>

1. Install the package and its peer dependencies:

   <Tabs>
   <TabList>
   <Tab>npm</Tab>
   <Tab>pnpm</Tab>
   <Tab>bun</Tab>
   </TabList>
   <TabPanels>
   <TabPanel>
   ```bash
   npm install -D eslint-config-setup eslint typescript
   ```
   </TabPanel>
   <TabPanel>
   ```bash
   pnpm add -D eslint-config-setup eslint typescript
   ```
   </TabPanel>
   <TabPanel>
   ```bash
   bun add -D eslint-config-setup eslint typescript
   ```
   </TabPanel>
   </TabPanels>
   </Tabs>

2. Create your ESLint config file:

   ```typescript
   // eslint.config.ts
   import { getConfig } from "eslint-config-setup"

   export default await getConfig({ react: true })
   ```

3. Run ESLint:

   ```bash
   npx eslint .
   ```

</Steps>

<Note>
Requires ESLint >= 9.22 and TypeScript >= 5.0.
</Note>

## What you get

That single `getConfig()` call gives you:

- Type-checked TypeScript with `strictTypeChecked`
- React 19 with Hooks, Compiler, and JSX-A11y
- Import cycle detection and auto-sorted imports
- Code quality analysis via SonarJS
- Security rules
- Spell checking in code
- JSON and Markdown linting
- Vitest and Playwright overrides
- Prettier compatibility

No plugin juggling. No copy-pasted config blocks.

## Configuration flags

Enable features by setting flags to `true`:

```typescript
export default await getConfig({
  react: true,   // React 19+, Hooks, JSX-A11y, Storybook, Testing Library
  node: true,    // Node.js globals and rules
  ai: true,      // Strict rules for AI-assisted development
  oxlint: true,  // Disable rules covered by OxLint
})
```

## Customizing rules

Every config is fully customizable:

```typescript
import { getConfig, disableRule, addRule } from "eslint-config-setup"

const config = await getConfig({ react: true, ai: true })

disableRule(config, "@typescript-eslint/no-magic-numbers", { scope: "tests" })
addRule(config, "no-console", "off", { scope: "scripts" })

export default config
```

See the [Rule API](/guide/rule-api) for the full set of manipulation functions.
